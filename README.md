# Siam ML Hack 2025 — AI Solution for Well Test Analysis

> **Конкурсное решение, разработанное в рамках хакатона Siam ML Hack 2025. Проект посвящён автоматизации анализа гидродинамических исследований скважин с использованием методов машинного обучения. Реализован полный цикл разработки ML-решения: от подготовки данных и обучения моделей до валидации и генерации итоговых прогнозов.**

---

# 📖 О проекте

**Siam ML Hack 2025** — проект, созданный для решения прикладной задачи анализа данных гидродинамических исследований (ГДИС) в рамках хакатона по искусственному интеллекту.

Основная цель проекта — автоматическое определение режимов фильтрации и прогнозирование соответствующих числовых характеристик на основе исходных данных исследований.

Ключевой особенностью решения является **двухэтапный подход (Classification → Regression)**:

- сначала модель определяет наличие каждого режима исследования;
- затем только для найденных режимов прогнозируются их числовые характеристики.

Такой подход позволяет значительно уменьшить количество ложных предсказаний и сделать модель более устойчивой к новым данным.

Проект охватывает полный цикл разработки ML-решения: подготовку данных, обучение моделей, кросс-валидацию, сохранение обученных моделей и применение их для обработки новых исследований.

# ⚙️ Pipeline решения

## 1. Подготовка данных

Перед обучением модели выполняются следующие этапы обработки данных:

- очистка датасета;
- удаление служебных признаков;
- обработка пропущенных значений;
- формирование признакового пространства;
- подготовка обучающих выборок.

---

## 2. Классификация режимов

Первый этап пайплайна определяет наличие каждого режима исследования.

Используется подход **Multi-Label Classification**, позволяющий одновременно прогнозировать несколько независимых бинарных признаков.

Именно этот этап определяет, для каких режимов необходимо выполнять дальнейший расчёт параметров.

---

## 3. Регрессионное прогнозирование

После завершения классификации запускается регрессионная модель.

Для каждого обнаруженного режима вычисляются соответствующие числовые характеристики.

Если классификатор определяет отсутствие режима, соответствующее значение автоматически устанавливается равным нулю.

Подобная архитектура значительно уменьшает количество ошибочных прогнозов.

---

## 4. Валидация модели

Во время разработки проводятся:

- стратифицированное разбиение выборки;
- кросс-валидация;
- оценка качества классификации;
- анализ точности регрессионной модели;
- сравнение различных конфигураций моделей.

---

## 5. Сохранение моделей

После обучения лучшие модели сохраняются для последующего использования.

В проекте сохраняются:

- модель классификации;
- модель регрессии.

Это позволяет применять решение без повторного обучения.

---

## 6. Инференс

При обработке новых исследований выполняются следующие шаги:

1. загрузка сохранённых моделей;
2. подготовка входных данных;
3. определение активных режимов;
4. прогнозирование числовых параметров;
5. формирование итогового результата.

# 📈 Оценка качества

Качество моделей оценивается с использованием нескольких подходов.

Особое внимание уделяется:

- точности классификации;
- качеству регрессионных прогнозов;
- устойчивости модели;
- способности к обобщению;
- воспроизводимости результатов.

В процессе разработки используются стандартные метрики **F1-score** и **MAE (Mean Absolute Error)**.

# 💡 Особенности решения

Главная особенность проекта — **каскадная архитектура машинного обучения**.

Вместо прогнозирования всех параметров одновременно задача разделяется на два независимых этапа:

- определение существования режима;
- прогнозирование его количественных характеристик.

Такой подход лучше соответствует логике предметной области, уменьшает влияние ложных срабатываний и повышает качество итогового решения.
